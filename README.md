# 🌌 Vibe Vault

Welcome to **Vibe Vault**, a curated public catalogue and review hub dedicated to smaller, atmospheric, and vibe-coded indie games. 

This platform is built to showcase games that prioritize mood, soundscapes, and aesthetic depth over mainstream mechanics—giving indie gems and micro-experiences a dedicated space to be discovered and discussed.

---

## 🛠️ Features

* **Aesthetic-First Curation:** Games categorized by specific atmospheric vibes (e.g., *#liminal, #neon-noir, #cozy-ambient*).
* **Community Review System:** Integrated interactive elements allowing the public to leave their own ratings and thoughts.
* **Blazing Fast & Lightweight:** Hosted entirely for free via GitHub Pages with minimal overhead and clean markdown-driven pipelines.

---

## 🚀 Getting Started & Local Setup

If you want to run this project locally on your machine or test visual changes before pushing them live:

1. **Clone the repository:**
   ```bash
   git clone https://github.com/Vibe-Vault/Vibe-Vault.git
   ```
2. **Navigate into the directory:**
   ```bash
   cd vibe-vault
   ```
3. **Open the project:**
   Open `index.html` in any modern web browser to preview the site locally.

---

## ✍️ How to Add a New Game Review

To keep the catalogue growing uniformly, use the following standard review template structure when adding entries to the core index:

1. **Prepare the Assets:** Place a high-quality header image or gameplay GIF into the `/assets/images/` directory.
2. **Update the Catalogue:** Structure your review with this spatial hierarchy:
   * **Hero Visual:** Top-aligned mood piece or looping GIF to establish the visual tone.
   * **Vibe Tags:** 3 to 4 hyper-specific aesthetic descriptors (e.g., *#cozy-isometric, #dreampunk*).
   * **The Critique:** A tight, impactful 150–250 word focus on atmospheric execution and how the game feels.
3. **Commit & Deploy:**
   ```bash
   git add .
   git commit -m "Add review for [Game Name]"
   git push origin main
   ```
   *Your updates will automatically compile and deploy to the live site via GitHub Pages within a few minutes.*
